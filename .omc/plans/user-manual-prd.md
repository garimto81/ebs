## 1. Executive Summary

### 1.1 제품 개요

EBS(Event Broadcasting System)는 RFID 기반 포커 카드 인식 및 실시간 방송 오버레이 시스템입니다. PokerGFX를 벤치마크로 하되, 오픈소스 기반의 비용 효율적인 솔루션을 목표로 합니다.

### 1.2 비전

| 항목 | PokerGFX (벤치마크) | EBS (목표) |
|------|---------------------|------------|
| 플랫폼 | .NET Framework (Windows) | Python + React (Cross-platform) |
| 라이선스 | 상용 (Basic/Pro/Enterprise) | 오픈소스 + 상용 옵션 |
| 하드웨어 | 전용 RFID 시스템 | ESP32 + MFRC522 (저비용) |
| 그래픽 엔진 | SkiaSharp | CSS/Canvas + OBS Browser Source |

### 1.3 핵심 가치 제안

1. **저비용**: 하드웨어 비용 $50 미만 (PokerGFX 대비 90% 절감)
2. **오픈소스**: 커스터마이징 및 확장 가능
3. **크로스 플랫폼**: Windows, macOS, Linux 지원
4. **방송 친화적**: OBS 네이티브 통합

**목표**: 기본 카드 인식 + 오버레이 시스템

**기간**: 4주

#### 3.2.1 하드웨어 레이어

```
목표: 안정적인 RFID 카드 인식

기능:
├── ESP32 펌웨어
│   ├── MFRC522 초기화 및 읽기
│   ├── JSON Serial 출력
│   └── 에러 핸들링
├── 카드 등록 도구
│   ├── UID 스캔
│   ├── 카드 매핑 (Suit + Rank)
│   └── DB 저장
└── 하드웨어 진단
    ├── 연결 상태 확인
    └── 읽기 성공률 측정
```

**성공 기준**:
- 카드 인식 시간 < 0.5초
- 인식 정확도 > 99%
- 52장 + 조커 2장 등록 완료

#### 3.2.2 서버 레이어

```
목표: 실시간 데이터 처리 및 WebSocket 브로드캐스트

기능:
├── FastAPI 서버
│   ├── Serial 수신 서비스
│   ├── 카드 DB 조회
│   └── WebSocket 브로드캐스트
├── REST API
│   ├── GET /api/cards - 카드 목록
│   ├── GET /api/status - 시스템 상태
│   └── POST /api/cards - 카드 등록
├── WebSocket Events
│   ├── card_detected - 카드 인식
│   ├── card_removed - 카드 제거
│   └── reader_status - 리더 상태
└── SQLite Database
    ├── cards 테이블
    └── sessions 테이블 (선택)
```

**성공 기준**:
- WebSocket 지연시간 < 100ms
- 동시 접속 10+ 클라이언트 지원
- 24시간 안정 운영

#### 3.2.3 프론트엔드 레이어

```
목표: OBS 호환 오버레이 UI

기능:
├── 오버레이 컴포넌트
│   ├── CardDisplay - 단일 카드 렌더링
│   ├── HoleCards - 홀카드 2장 (Hold'em)
│   ├── CommunityCards - 커뮤니티 5장
│   └── ConnectionStatus - 연결 상태
├── OBS 최적화
│   ├── 투명 배경 (Chroma Key 불필요)
│   ├── 고정 해상도 (1920x1080)
│   └── 저지연 렌더링
└── 기본 설정
    ├── 서버 URL 설정
    └── 카드 크기 조절
```

**성공 기준**:
- OBS Browser Source 정상 동작
- 카드 표시 지연 < 200ms (인식→표시)
- 투명 배경 렌더링

#### 3.2.4 Secure Delay (핵심 기능)

```
목표: 라이브 스트리밍 보안

기능:
```

## 5. 상세 요구사항

### 5.1 MVP 기능 명세

#### 5.1.1 FR-001: RFID 카드 인식

```yaml
ID: FR-001
이름: RFID 카드 인식
우선순위: P0 (필수)
Phase: MVP

설명: |
  RFID 태그가 부착된 포커 카드를 인식하고
  카드 정보(Suit, Rank)를 반환한다.

입력:
  - RFID 카드 (13.56MHz, MIFARE)

출력:
  - JSON: { uid, suit, rank, display }

성공 기준:
  - 인식 시간 < 500ms
  - 정확도 > 99%
  - 중복 인식 방지

의존성:
  - ESP32 펌웨어
  - MFRC522 라이브러리
  - 카드 DB
```

#### 5.1.2 FR-002: 실시간 WebSocket 통신

```yaml
ID: FR-002
이름: 실시간 WebSocket 통신
우선순위: P0 (필수)
Phase: MVP

설명: |
  카드 인식 이벤트를 실시간으로 연결된
  모든 클라이언트에 브로드캐스트한다.

이벤트:
  card_detected:
    - uid: string
    - card: { suit, rank, display }
    - timestamp: number
    - reader_id: number

  card_removed:
    - reader_id: number
    - timestamp: number

  reader_status:
    - connected: boolean
    - port: string
    - readers: number

성공 기준:
  - 지연시간 < 100ms
  - 동시 접속 10+ 클라이언트
  - 재연결 자동 처리
```

#### 5.1.3 FR-003: OBS 오버레이 UI

```yaml
ID: FR-003
이름: OBS 오버레이 UI
우선순위: P0 (필수)
Phase: MVP

설명: |
  OBS Browser Source로 사용 가능한
  투명 배경의 카드 오버레이를 제공한다.

컴포넌트:
  HoleCards:
    - 카드 2장 (Hold'em)
    - 위치 조절 가능
    - 크기 조절 가능

  CommunityCards:
    - 카드 5장 (Flop/Turn/River)
    - 가로 배치
    - 애니메이션 (Phase 2)
```
