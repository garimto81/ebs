## 조사 결과 요약

### 공식 API 존재 여부

**결론: 공개 개발자 API 미확인**

- WSOP.com, WSOP+ 앱에 공개 API 문서 없음
- poker.tv, GGPoker 등 파트너사 전용 데이터 피드 가능성 있음
- 방송사(ESPN, PokerGO)는 별도 계약 추정

## 필요 데이터 포인트

EBS가 실시간으로 필요한 데이터:

| 데이터 | 우선순위 | Stage 0-1 대응 | Stage 2+ 연동 가능성 |
|--------|----------|----------------|---------------------|
| **플레이어명** | P0 | 수동 입력 | API 또는 CSV import |
| **칩 카운트** | P0 | 수동 입력 | 실시간 API 연동 |
| **현재 액션** | P1 | EBS 자체 기록 | WebSocket 연동 |
| **핸드 히스토리** | P1 | EBS 자체 기록 | 배치 API |
| **대회 정보** (토너먼트명, 블라인드) | P2 | 수동 입력 | CSV import |
| **팟 크기** | P2 | 계산 (칩 카운트 기반) | API 연동 |

## 방송 업계 일반적 연동 방식

### 1. 수동 입력 (가장 일반적)
- **현실:** ESPN, PokerGO도 초기에는 수동 입력 사용
- **장점:** 외부 의존성 없음, 즉시 구현 가능
- **단점:** 오타, 지연, 인력 소요

### 2. 파트너 데이터 피드 (대형 방송사 전용)
- **계약 형태:** NDA + 데이터 라이센스 계약
- **비용:** 예상 $10K-100K/년 (추정)
- **필요 조건:** WSOP 공식 파트너십 체결

### 3. 스크린 캡처/OCR (비공식)
- **기술:** 화면 캡처 → OCR → 데이터 추출
- **위험:** WSOP 이용약관 위반 가능성
- **권장하지 않음**

## 3가지 연동 시나리오

### 시나리오 1: 완전 수동 입력 ✅ (Stage 0-1 권장)

**흐름:**
```
대회 시작 전:
  1. 플레이어 명단 수동 등록 (CSV 업로드 기능)
  2. 시작 칩 카운트 입력

대회 중:
  3. 핸드별 액션은 EBS가 RFID로 자동 감지
  4. 칩 카운트는 수동 업데이트 (또는 계산)
```

**장점:**
- Stage 0-1에서 즉시 구현 가능
- 가장 안정적
- 외부 의존성 없음

**단점:**
- 운영자 부담 증가
- 실시간 업데이트 어려움

**예상 구현 시간:** 2주

### 시나리오 2: CSV Import + 반자동 ✅ (Stage 1)

**흐름:**
```
대회 시작 전:
  1. WSOP 공식 명단 CSV 다운로드
  2. EBS에 일괄 import

대회 중:
  3. EBS가 RFID로 핸드 히스토리 자동 기록
  4. 칩 카운트는 수동 업데이트 (간헐적)
```

**구현 계획:**
- CSV 포맷 정의 (player_id, name, starting_chips)
- 업로드 UI 추가
- 검증 로직 (중복, 누락 체크)

**예상 구현 시간:** 1주

### 시나리오 3: 파트너 API 연동 ⚠️ (Stage 2+, 협의 필요)

**전제 조건:**
- WSOP Production 팀과 파트너십 체결
- API 스펙 문서 제공받음
- 데이터 사용 계약 체결

**연동 포인트:**
- **RESTful API**: 플레이어 정보, 대회 정보
- **WebSocket**: 실시간 칩 카운트, 액션 피드
- **배치 API**: 핸드 히스토리 export

**구현 난이도:** 높음 (API 스펙에 따라 2-6주)

**비용 추정:**
- 연간 라이센스: $10K-100K
- 개발 비용: $5K-20K

## Stage 2 로드맵 권장사항

### 1단계: 담당자 컨택 (Q2 2027)
- WSOP Production Manager 이메일
- EBS 데모 영상 제공
- 데이터 파트너십 제안

### 2단계: 요구사항 협의 (Q3 2027)
- 필요 데이터 포인트 명시
- 업데이트 주기 (실시간 vs 배치)
- SLA 요구사항 (99.9% uptime)

### 3단계: 파일럿 테스트 (Q4 2027)
- 1개 토너먼트 시범 연동
- 성능 검증 (레이턴시 < 500ms)
- 데이터 정확도 검증

### 4단계: 본격 연동 (2028+)
- 전체 WSOP 이벤트 적용
- 실시간 대시보드 구축
- 자동화 워크플로우 완성

## 리스크 및 대응책

| 리스크 | 확률 | 영향도 | 대응책 |
|--------|------|--------|--------|
| WSOP API 비공개 유지 | 높음 | 중 | 시나리오 1/2로 대응 |
| 파트너십 계약 실패 | 중 | 중 | CSV import로 우회 |
| API 비용 과다 | 낮음 | 중 | ROI 분석 후 협상 |
| API 장애 발생 | 낮음 | 높음 | Fallback to 수동 입력 |

## 대안 기술

WSOP 공식 API 불가 시:

### 1. Poker Hand History 표준 포맷
- **PokerStars HH Format**: 업계 표준
- **구현:** EBS가 자체 HH 생성 → 타사 툴 연동

### 2. Open Source Poker DB
- **PokerTracker**, **Holdem Manager** 포맷 지원
- **장점:** 에코시스템 활용

### 3. 커스텀 Data Layer
- EBS 자체 DB 스키마
- 외부 의존성 최소화
