## 위험 요소 (Risks)

### 1. 화면 전환 로직 복잡도
6개 화면이 단일 HTML 파일 내 탭으로 구현되어, 각 화면 간 상태 공유(현재 좌석, 게임 상태, 카드 선택)가 필요. JavaScript 상태 관리 없이 순수 DOM 조작만으로 구현 시 인터랙션 불완전 위험.

**완화**: 최소한의 전역 `state` 객체를 선언하고, 버튼 클릭 핸들러가 해당 객체를 갱신하도록 설계. 와이어프레임 수준에서 과도한 로직 구현 금지.

### 2. 동적 버튼 전환 오해 가능성
CALL/RAISE-TO vs CHECK/BET 전환이 와이어프레임에서 어느 한 상태로 고정될 경우, 검토자가 단일 상태로 오해할 위험.

**완화**: 메인 액션 화면(002135 - CALL/RAISE-TO 상태)과 플랍 액션 화면(002217 - CHECK/BET 상태)을 별도 탭으로 분리하여 두 상태를 모두 표현.

### 3. 색상 정확도
PokerGFX의 진녹색(#4a7c59 계열), 진빨강(#8b0000 계열), 금색(#c8a030 계열), 진회색(#2a2a2a) 계열은 스크린샷 픽셀 분석 없이 정확한 hex 값 추정이 어려움.

**완화**: 스크린샷 관찰 기반 근사 색상 사용. 최종 구현(Flutter) 단계에서 정밀 색상 추출 진행.

### 4. 단일 파일 용량
6화면 × 상세 HTML + 인라인 카드 그리드(52장) = 1,000~2,000 라인 예상. 파일이 과도하게 커질 경우 유지보수 난항.

**완화**: CSS는 `<style>` 태그에 공통 클래스 집약, 반복 구조는 JavaScript `innerHTML` 생성으로 처리.

### TASK-05: 화면 2 — 게임 진입/메인 액션 화면 구현 (CALL/RAISE-TO 상태)

**설명**: 본방송 중 핵심 운영 화면. 베팅이 있는 상태(CALL/RAISE-TO).

**수행 방법**:
- TASK-02 툴바 + TASK-03 좌석 행
- 중앙 3분할 레이아웃 (CSS Grid `grid-template-columns: auto 1fr auto`):
  - 좌측: MISS DEAL 버튼 (빨강, 2행 텍스트, 큰 크기)
  - 중앙: 플레이어 이름 입력 박스(상단) + 상태 바 `(1) SEAT 1 - STACK 1,000,000`(하단) + 주황 원(⚫)
  - 우측: ★ TAG 버튼(상단) + HIDE GFX 버튼(하단, 은회색)
- 하단 액션 버튼 행 (5개): ←(UNDO), FOLD(진다크레드), CALL(진녹색), RAISE-TO(중간녹색), ALL IN(회색)
- `data-state="call"` 속성으로 상태 표시

**Acceptance Criteria**:
- MISS DEAL 버튼 크기가 다른 버튼보다 현저히 크게 표시
- 하단 5개 버튼이 전체 너비에 균등 배치
- 상태 바 텍스트 중앙 정렬
- HIDE GFX는 우측 고정, 항상 visible

### TASK-06: 화면 3 — 카드 선택 그리드 구현

**설명**: 52장 카드 수동 선택 그리드. RFID 실패 시 사용.

**수행 방법**:
- 배경 #0a0a0a
- 상단 바: ◀(버튼, 진녹색), 선택 카드 슬롯 5개, OK 버튼(진녹색, 우측)
- 카드 그리드: `display: grid; grid-template-columns: repeat(13, 1fr); grid-template-rows: repeat(4, 1fr)`
- 행 순서: ♠(스페이드), ♥(하트), ♦(다이아), ♣(클럽)
- 열 순서: A, K, Q, J, 10, 9, 8, 7, 6, 5, 4, 3, 2
- 각 카드 셀: 흰 배경 또는 카드 색상(빨강/검정), 텍스트 표시(예: `A♠`, `7♥`)
- 클릭 이벤트: 선택 시 상단 슬롯에 추가, `.selected` 클래스로 반투명 처리
- 7♥, 6♠ 초기 선택 상태 시뮬레이션

**Acceptance Criteria**:
- 52장이 4행 × 13열로 정확히 배치
- 이미 선택된 카드(7♥, 6♠)는 반투명(opacity 0.4) 또는 회색 처리
- ◀ 클릭 시 마지막 선택 카드 취소
- OK 버튼은 진녹색, 우측 상단 고정

### TASK-07: 화면 4 — 플랍 액션 화면 구현 (CHECK/BET 상태)

**설명**: 화면 2와 동일한 레이아웃이지만 커뮤니티 카드 표시 + CHECK/BET 버튼.

**수행 방법**:
- 화면 2 HTML 구조를 복사하되 변경점만 적용:
  - 중앙 영역 상단: 이름 입력 박스 대신 카드 표시 박스 `[ 7♥ ][ 6♠ ][ 4♥ ][   ][   ]`
  - 액션 버튼: CALL → CHECK, RAISE-TO → BET
  - 좌석 상태: SEAT 1=회색(fold), SEAT 2=노란(action-on), SEAT 3=빨강
- `data-state="check"` 속성

**Acceptance Criteria**:
- 커뮤니티 카드 박스 3장이 채워져 있고 2장은 빈 상태로 표시
- CHECK/BET 버튼 텍스트가 화면 2의 CALL/RAISE-TO와 명확히 구분
- SEAT 1이 폴드 상태(진한 회색)로 표시

**수행 방법**:

- 좌측 통계 테이블: `<table>` 또는 CSS Grid, 7컬럼(SEAT/STACK/VPIP%/PFR%/AGRFq%/WTSD%/WIN), 10행(SEAT 1~10)
- STACK 헤더 셀: 배경 #c8a030(금색)
- 각 데이터 셀: 배경 #333, border-radius 6px, padding, 텍스트 흰색
- 활성 데이터(SEAT 1~3): 실제 숫자 표시, 나머지 빈 상태
- 우측 방송 제어 패널(세로 Flex):

  - GFX 버튼 (금색 - 활성)

  - REMAIN + TOTAL 버튼 행

- STACK 컬럼 헤더가 금색 배경으로 구분
- GFX 버튼이 LIVE 버튼보다 시각적으로 강조(금색)

- CANCEL 버튼이 진빨강 그라디언트 배경으로 표시
