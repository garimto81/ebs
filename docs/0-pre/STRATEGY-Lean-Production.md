# EBS 전략: 소수 인원 운영 프로덕션

**Version**: 1.0.0
**Date**: 2026-02-02
**Status**: Draft
**Core Principle**: 소수의 인원으로 운영 가능한 프로덕션

---

## Executive Summary

> **"1-2명이 운영 가능한 프로덕션 시스템"**

EBS의 최종 목표는 **현재 5-6명이 필요한 WSOP STUDIO 방송을 1-2명이 운영**할 수 있도록 만드는 것입니다.
이를 위해 5개 Layer의 체계적 전략을 수립합니다.

## Part 1: 현재 운영 인력 분석

| **데이터 입력** | 1명 | 칩 카운트, 통계 업데이트 | 100% 수동 |
| **백업/지원** | 1명 | 교대, 이슈 대응 | 대기 |
| **총계** | **5-6명** | | |

## Part 3: Layer 2 - GFX/WSOP+ 기능 분류 (핵심)

### 3.1 왜 Layer 2가 중요한가?

> **기능 분류를 통해 "무엇을 자동화할 것인가"를 결정합니다.**

PokerGFX 54개 기능을 3가지 카테고리로 분류:

| 카테고리 | 데이터 소스 | 자동화 가능성 | 예시 |
|----------|------------|:------------:|------|
| **A. EBS 독립** | RFID + 자체 DB | Stage 1에서 자동 | 카드 인식, 승률 계산 |
| **B. WSOP+ 연동** | WSOP+ API | Stage 2에서 자동 | 플레이어 정보, 칩 카운트 |
| **C. 수동 유지** | 운영자 입력 | 자동화 불가/불필요 | 게임 타입 선택, 특수 상황 |

## Part 5: Stage별 자동화 상세

### 5.1 Stage 1: 기반 자동화 (인력 -1명)

**목표**: Card Scanner 역할 제거

| 자동화 항목 | 구현 방법 | 효과 |
|------------|----------|------|
| 카드 인식 | RFID + ST25R3911B | 스캔 인력 불필요 |
| 승률 계산 | Monte Carlo 실시간 | 계산 자동화 |
| 핸드 히스토리 | DB 자동 저장 | 기록 자동화 |
| Trustless Mode | 30초 딜레이 버퍼 | 보안 자동화 |

**남은 수동 작업**:
- 액션 입력 (Fold/Call/Raise/Bet)
- 플레이어 정보 입력 (이름, 칩)
- 칩 카운트 갱신 (브레이크)
- 하이라이트 태깅

### 5.2 Stage 2: 데이터 자동화 (인력 -1명)

**목표**: 데이터 입력 역할 제거

| 자동화 항목 | 데이터 소스 | 효과 |
|------------|------------|------|
| 플레이어 이름 | WSOP+ API | 입력 불필요 |
| 초기 칩 스택 | WSOP+ API | 입력 불필요 |
| 칩 카운트 (브레이크) | WSOP+ 동기화 | 갱신 불필요 |
| 플레이어 통계 | WSOP+ 데이터 | 계산 불필요 |
| 토너먼트 정보 | WSOP+ API | 설정 최소화 |

**남은 수동 작업**:
- 액션 입력 (여전히 수동)
- 특수 상황 처리 (미스딜, 팟 조정)
- 하이라이트 태깅

### 5.3 Stage 3: 운영 자동화 (인력 -1~2명)

**목표**: GFX Operator 최소화

| 자동화 항목 | 구현 방법 | 효과 |
|------------|----------|------|
| 팟 자동 계산 | 액션 기반 연산 | 계산 불필요 |
| 칩 실시간 추적 | 베팅/승리 반영 | 갱신 불필요 |
| 액션 제안 | 패턴 분석 | 입력 80% 감소 |
| 핸드 종료 감지 | 칩 이동 인식 | 클릭 불필요 |
| 딜러 버튼 이동 | 자동 회전 | 클릭 불필요 |

**남은 수동 작업**:
- 예외 상황 처리 (5-10%)
- 액션 확인/수정 (20%)

### 5.4 Post-Stage 3: AI 자동화 (PD 부담 감소)

**목표**: 콘텐츠 생산 자동화

| 자동화 항목 | AI 기능 | 효과 |
|------------|---------|------|
| 하이라이트 자동 태깅 | 빅팟/블러프 감지 | 태깅 자동화 |
| 자동 클립 생성 | 핸드 데이터 기반 | 편집 최소화 |
| 실시간 해설 지원 | 통계 기반 문구 | 해설 보조 |
| 스위칭 트리거 | 액션 감지 | TD 부담 감소 |

| 운영자 1명 이탈 | 고 | 백업 운영자 교육 필수 |

| 자동화 정확도 부족 | 중 | 제안 모드 (확인 필수) |
