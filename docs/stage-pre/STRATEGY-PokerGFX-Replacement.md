## 1. Executive Summary

> **"GFX가 없어진다면, 무엇을 준비해야 하는가?"**

본 문서는 PokerGFX 서비스 종료/이탈 시나리오에 대비한 전략적 준비 사항을 정의합니다.

## 2. PokerGFX 종료 시나리오

### 2.1 시나리오 정의

| 시나리오 | 설명 | 확률 | 준비 기간 |
|----------|------|:----:|----------|
| **A. SW 라이선스 종료** | 소프트웨어만 사용 불가, 하드웨어 유지 | 중 | 3-6개월 |
| **B. HW+SW 중단** | Reader Module 공급 중단 포함 | 중 | 6-12개월 |
| **C. 완전 서비스 종료** | 기술 지원 완전 중단 | 저 | 12개월+ |
| **D. 비용 문제** | 라이선스 비용 상승으로 경제성 상실 | 고 | 즉시 결정 |

### 2.2 대응 전략

| 시나리오 | 대응 전략 |
|----------|----------|
| A | EBS SW만 자체 개발, 기존 HW 재사용 |
| B | EBS Full Stack (HW+SW) 자체 구축 |
| C | EBS 완전 독립 시스템 구축 |
| D | EBS로 점진적 마이그레이션 |

## 3. PokerGFX 핵심 기능 분석

### 3.1 필수 기능 (P0 - 방송 불가 시)

| Category | Feature | PokerGFX 구현 | EBS 복제 우선순위 |
|----------|---------|--------------|:----------------:|
| **RFID 인식** | 실시간 카드 UID 읽기 | 다중 안테나 + Reader Module | **P0** |
| **카드 매핑** | UID → 카드 정보 변환 | AES-256 암호화 DB | **P0** |
| **Viewer Overlay** | 홀카드, 팟, 보드 표시 | OBS Browser Source | **P0** |
| **Action Tracker** | 베팅 액션 수동 입력 | Windows App | **P0** |
| **팟 계산** | 액션 기반 자동 계산 | 내장 로직 | **P0** |

### 3.2 부가 기능 (P1 - 있으면 좋음)

| Feature | 가치 | EBS Stage |
|---------|------|:---------:|
| **플레이어 통계** | VPIP, PFR, 3Bet% 등 | Stage 1 |
| **Hand History** | 과거 핸드 검색/재생 | Stage 1 |
| **Equity Calculator** | 실시간 승률 계산 | Stage 1 |
| **Leaderboard** | 칩 순위 표시 | Stage 1 |
| **Custom Skins** | 브랜딩 커스터마이징 | Stage 1 |

### 3.3 보안 기능

| Feature | PokerGFX 구현 | 필수 여부 | EBS Stage |
|---------|--------------|:---------:|:---------:|
| **Secure Delay** | Video → Delay → Graphic | **필수** | Stage 1 |
| **Trustless Security** | 카드 정보 딜레이 후 생성 | 권장 | Stage 1 |
| **TLS 1.2+ 통신** | USB-to-Reader 암호화 | 권장 | Stage 1 |
| **AES-256 DB** | 카드 매핑 DB 보호 | **필수** | Stage 1 |

## 5. 기존 GFX 개선 희망 기능

> PokerGFX에 있었으면 했던 기능들 - EBS에서 구현 계획

### 5.1 사용자 경험 개선

| Pain Point | 현재 GFX | EBS 개선 | Stage |
|------------|----------|----------|:-----:|
| 복잡한 운영 인터페이스 | 전문 인력 필요 | **원클릭 Quick Actions** | 1 |
| 느린 기술 지원 (72시간+) | 글로벌 지원 | **로컬 지원 + 자체 역량** | 1 |
| 아시아 시장 미지원 | 영어만 | **한글 UI, 한국 시간대** | 1 |
| Stream Deck 미지원 | 추가 장치 불가 | **Stream Deck API 연동** | 2 |
| 음성 입력 미지원 | 수동 입력만 | **음성 인식 액션 입력** | 3 |

### 5.2 자동화 확대

| Pain Point | 현재 GFX | EBS 개선 | Stage |
|------------|----------|----------|:-----:|
| 모든 액션 수동 입력 | 운영자 필수 | **스마트 베팅 예측** | 1 |
| 칩 카운트 수동 | 브레이크 입력 | **WSOP+ 자동 동기화** | 2 |
| 플레이어 정보 수동 | 매번 입력 | **CSV Import / API Pull** | 2 |
| 하이라이트 수동 | PD 수동 태깅 | **빅팟/블러프 자동 감지** | 3 |
| 핸드 종료 수동 | 운영자 클릭 | **자동 감지 (칩 이동)** | 3 |

### 5.3 데이터 활용 강화

| Pain Point | 현재 GFX | EBS 개선 | Stage |
|------------|----------|----------|:-----:|
| 데이터 외부 종속 | GFX DB에 종속 | **자체 DB 완전 소유** | 0-1 |
| 제한된 통계 | 기본 VPIP/PFR | **커스텀 통계 무제한** | 1 |
| Export 제한 | 제한적 내보내기 | **SQL 직접 쿼리, API** | 1 |
| 콘텐츠 수동 제작 | 리포트 수동 | **자동 하이라이트 생성** | 3 |

### 5.4 커스터마이징 확대

| Pain Point | 현재 GFX | EBS 개선 | Stage |
|------------|----------|----------|:-----:|
| 스킨 제한 | 제공 스킨만 | **완전 커스텀 스킨** | 1 |
| 레이아웃 고정 | 정해진 위치 | **드래그 앤 드롭 에디터** | 2 |
| 애니메이션 제한 | 기본만 | **CSS/Lottie 커스텀** | 2 |
| 브랜딩 제한 | 로고 교체 | **전체 UI 테마 커스텀** | 2 |

## 6. WSOP+ 데이터 활용 방안

### 6.1 수집 가능 데이터 (추정)

| Category | Data Points | 수집 시점 |
|----------|-------------|----------|
| **플레이어 정보** | 이름, 국적, 시트, 프로필 | 등록 시 |
| **칩 카운트** | 현재 칩, 리바이/애드온 | 실시간 or 브레이크 |
| **토너먼트 정보** | 이벤트명, 블라인드, 페이아웃 | 생성 시 |
| **플레이어 전적** | WSOP 대회 기록, 수상 내역 | 요청 시 |
| **핸드 히스토리** | 과거 방송 핸드 | 배치 |

### 6.2 EBS 활용 방안

| Use Case | 입력 | 출력 | Stage |
|----------|------|------|:-----:|
| **자동 플레이어 설정** | 플레이어 목록 | Pre-Start 자동 채움 | 2 |
| **칩 자동 보정** | 브레이크 칩 카운트 | 데이터 검증/수정 | 2 |
| **해설 보조 통계** | 플레이어 전적 | 오버레이 표시 | 2 |
| **시청자 정보 강화** | 프로필, 국적 | Player HUD 강화 | 2 |
| **콘텐츠 자동화** | 핸드 히스토리 | 하이라이트 자동 생성 | 3 |

### 6.3 연동 고려사항

| Category | 고려사항 | 대응 방안 |
|----------|----------|----------|
| **API 접근** | 공개 API 없음 | 파트너십 체결 필요 |
| **데이터 소유권** | WSOP가 마스터 | 계약서 사용 범위 명시 |
| **개인정보** | 플레이어 동의 필요 | GDPR/개인정보법 준수 |
| **SLA** | API 가용성 | 폴백 모드 구현 (수동 입력) |
| **비용** | 라이선스 비용 | $10K-100K/년 예상 |

## 7. 마이그레이션 로드맵

### 7.1 전환 타임라인

```
2026 Q1 (Stage-Pre)
├── RFID 모듈/공급원 확정 [완료]
├── 기술 설계 문서화 [완료]
└── UI 목업 작성 [완료]

2026 Q2-Q3 (Stage 0)
├── 하드웨어 구매 및 조립
├── ESP32 + MFRC522 통신 검증
└── E2E PoC 완성

2026 Q4 (Stage 1)
├── PokerGFX P0 기능 복제
├── Viewer Overlay + Action Tracker
├── Secure Delay 구현
└── 병행 운영 테스트

2027 (Stage 2)
├── WSOP+ API 파트너십
├── 데이터 연동 구현
└── GFX → EBS 전환 완료

2028 (Stage 3)
├── 80% 자동화
└── EBS 독창 기능
```

### 7.2 병행 운영 전략

| Phase | 기간 | GFX 역할 | EBS 역할 |
|-------|------|----------|----------|
| **Alpha** | Stage 0 | 메인 | 테스트 환경 |
| **Beta** | Stage 1 초반 | 메인 | 비공개 테스트 |
| **Parallel** | Stage 1 후반 | 백업 | 일부 방송 투입 |
| **Cutover** | Stage 2 | 비상용 | 메인 |
| **Retired** | Stage 3 | 사용 안함 | 유일 시스템 |
