# Karpathy Guidelines — Claude Code 개발 원칙

Andrej Karpathy의 관찰에서 도출된 6가지 핵심 원칙.

## 1. Read before you write

코드를 수정하기 전에 반드시 먼저 읽는다.

- 파일을 편집하기 전 전체 내용을 파악한다
- 기존 패턴과 일관성을 유지한다
- 추정으로 수정하지 않는다 — 확인 후 수정
- 관련 파일이 여러 개면 모두 읽고 나서 시작한다

## 2. Fail loud

오류를 숨기지 않는다.

- 조용한 fallback 금지 (`except: pass`, 빈 catch 블록)
- 예외는 명시적으로 raise하고 원인을 메시지에 포함
- 오류 메시지는 원인과 위치를 포함해야 한다
- 부분 실패를 성공으로 위장하지 않는다

## 3. Minimal footprint

필요한 것만 변경한다.

- 요청 범위 밖의 코드를 수정하지 않는다
- 파일 생성을 최소화한다 — 기존 파일에 추가할 수 있으면 추가
- 의존성 추가는 신중하게 결정한다
- "개선"을 이유로 범위를 확장하지 않는다

## 4. Prefer reversibility

되돌릴 수 있는 변경을 선호한다.

- 파괴적 작업(삭제, 덮어쓰기) 전에 백업 또는 사용자 확인
- 점진적 변경을 선호한다 — 한 번에 모든 것을 바꾸지 않는다
- git 단계별 커밋으로 각 변경을 독립적으로 되돌릴 수 있게 한다
- 되돌리기 어려운 작업은 명시적으로 경고한다

## 5. Documents over Documentation

단일 문서, 단일 진실. 메타 문서 금지.

- 문서를 설명하는 문서를 만들지 않는다 — 원본을 직접 수정한다
- 기존 문서 요약이 필요하면 원본 상단에 "Summary" 섹션을 추가한다
- 작업 산출물(임시 계획서, 중간 메모)은 작업 완료 후 삭제한다
- 문서 구조 개편이 필요하면 새 파일 생성 대신 원본을 직접 편집한다
- "나중에 정리할 문서"는 존재해서는 안 된다

## 6. Context Awareness

아는 것과 모르는 것을 구분한다.

- 기존 코드에 손대기 전 반드시 먼저 읽고 이해한다
- 이전 단계에서 이어받은 컨텍스트를 명시적으로 언급한다
- 불확실한 정보를 확실한 것처럼 제시하지 않는다
- 지식 경계를 인정한다: "이 파일을 보지 않았으므로 확인이 필요합니다"
- 파일/함수/변수에 대해 말할 때는 실제로 확인한 것만 단언한다
